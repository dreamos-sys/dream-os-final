# M-STANDARDS (Inspired by Google Engineering Practices)

1. **The Rule of 500**: Jika script atau modul melebihi 500 baris, pecah (Decompose)!
2. **Performance Budget**: Jangan menambah aset tanpa mengukur dampaknya pada LCP.
3. **Hyrum's Law**: Hati-hati dengan interface API, semua perilaku (termasuk bug) akan diandalkan oleh user.
4. **Beyonce Rule**: "If you liked it, you should have put a test on it." Tidak ada fitur tanpa test!
5. **Source-Driven**: Grounding semua keputusan pada dokumentasi MDN, Web.dev, atau Repo resmi framework.
