# Karpathy Guidelines for Dream OS

Behavioral guidelines to reduce common LLM coding mistakes.

## Principles
1. **Think Before Coding** — State assumptions, ask if uncertain
2. **Simplicity First** — Minimum code that solves the problem
3. **Surgical Changes** — Touch only what you must
4. **Goal-Driven Execution** — Define verifiable success criteria

## Dream OS Context
- Vanilla JS only (no build tools)
- Target: Termux + Android browser + GitHub Pages
- Keep file count ≤3 main files
- Preserve spiritual elements (Bismillah, Shalawat)

## Success Criteria Examples
- `typeof openCommandCenter === 'function'`
- `openCommandCenter()` → UI appears
- Git diff shows only intended changes
